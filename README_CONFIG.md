# Конфигурация GRASP

Файл конфигурации — `config.php` (или `config.yaml`), возвращает массив. Все ключи опциональны — используются значения по умолчанию из `App::getDefaultConfig()`.

---

## database

| Ключ | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `driver` | string | `sqlite` | Только `sqlite`. |
| `host` | string | `/opt/grasp/db/grasp.sqlite` | Путь к файлу SQLite. |

|------|-----|-------------|----------|
| `path` | string | `/opt/grasp/storage` | Директория для хранения клонов репозиториев. |

|------|-----|-------------|----------|

## cron

Настройки планировщика (`cron.php`).

| Ключ | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `lock_file` | string | `/tmp/grasp_cron.lock` | Путь к lock-файлу, предотвращающему параллельный запуск cron. |
| `lock_timeout` | int | `300` | Время в секундах, после которого lock-файл считается устаревшим. |
| `lock_check_pid` | bool | `true` | Если `true` — при превышении `lock_timeout` проверяется, жив ли процесс-владелец lock-файла (через `/proc/{pid}` на Linux, `kill -0` как fallback). Это предотвращает снятие lock-а, если git clone/fetch всё ещё выполняется. Отключите (`false`), если нет прав на чтение `/proc` или используется не Linux. |
| `max_per_run` | int | `3` | Максимальное количество репозиториев, обрабатываемых за один запуск cron. |
| `retry_delay` | int | `300` | Задержка в секундах перед повтором после неудачного клонирования/обновления. |

## features

Включение/отключение опциональных возможностей.

| Ключ | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `deferred_delete` | bool | `false` | Если `true` — репозитории помечаются как `deferred_delete` вместо мгновенного удаления с диска. |

## http_timeout

| Тип | По умолчанию | Описание |
|-----|-------------|----------|
| int | `30` | Таймаут HTTP-запросов к API (GitHub/GitLab) в секундах. |

---

## timezone

| Тип | По умолчанию | Описание |
|-----|-------------|----------|
| string | `Europe/Moscow` | Часовой пояс для временных меток. |

---

## default_update_interval

| Тип | По умолчанию | Описание |
|-----|-------------|----------|
| string | `7d` | Интервал обновления по умолчанию для новых репозиториев. Формат: число + суффикс (`m` — минуты, `h` — часы, `d` — дни). Примеры: `30m`, `2h`, `7d`. |

---

## git

Настройки Git-операций.

| Ключ | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `binary` | string | `/usr/bin/git` | Путь к исполняемому файлу git. |
| `timeout` | int | `300` | Таймаут выполнения git-команд (clone/fetch) в секундах. |

---

## github

Настройки для GitHub API.

| Ключ | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `api_base` | string | `https://api.github.com` | Базовый URL GitHub REST API v3. |
| `api_timeout` | int | `15` | Таймаут запросов к GitHub API в секундах. |
| `web_base` | string | `https://github.com` | Базовый URL веб-интерфейса GitHub. |
| `max_retries` | int | `3` | Количество повторов при ошибках API. |
| `token` | string | `''` | Personal Access Token для GitHub API (повышает лимит запросов). |

---

## logging

Включение лог-файлов.

| Ключ | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `main` | bool | `true` | Логирование основных событий (запросы к API, ошибки). |
| `database` | bool | `false` | Логирование SQL-запросов. |
| `cron` | bool | `false` | Логирование работы планировщика. |

---

## access

IP-адреса и подсети для доступа к веб-интерфейсу и API.

| Ключ | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `admin_ips` | array | `['127.0.0.1', '192.168.111.1/24', '::1']` | Список IP/подсетей с полным доступом (администрирование). |
| `view_ips` | array | `['192.168.111.1/24', '0.0.0.0/0', '::/0']` | Список IP/подсетей с доступом только на чтение. `0.0.0.0/0` и `::/0` — открытый доступ. |

---

## git_http_backend

Настройка Git HTTP Backend (доступ к репозиториям по HTTP, см. `README_GIT_BACKEND.md`).

| Ключ | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `enabled` | bool | `false` | Включить Git HTTP Backend. |
| `info_ref_auto_update` | bool | `false` | Автоматически обновлять `info/refs` после clone/update. |
| `base_url` | string | `http://grasp.local/git` | Публичный URL, показываемый в интерфейсе для клонирования. |

---

## docker/config.docker.php

При запуске в Docker используйте `config.docker.php` (лежит рядом с `docker-compose.yml`). Его пути указывают на `/var/opt/grasp/...` — именно туда пробрасываются тома в `docker-compose.yml`.
